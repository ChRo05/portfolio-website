# 基本設計書 v0.1 — サイト構成 ＆ ワイヤーフレーム

更新日: 2026-06-24 / オーナー: KIKUCHI, Toru
前提: requirements-v0.2.md（要件定義）/ 実績内容は works-triage.md 記入後に流し込み

---

## 0. 設計の前提（確定事項）

| 項目 | 決定 |
|---|---|
| ページ形式 | ハイブリッド: トップは1枚LP / 実績だけ別ページ詳細 |
| 技術 | 素のHTML/CSS/JS（ビルド不要）。実績は JSON とビューを分離 |
| 公開 | GitHub Pages |
| 連絡 | SNS / GitHub / note のみ（メール・フォーム無し） |
| 主軸 | クライアント目線・30秒で伝わる・入口易しく奥は技術的 |

---

## 1. サイトマップ

```
/                 トップLP（ヒーロー〜連絡まで縦1枚）
└── /works/[id]   各実績の詳細ページ（課題→対応→成果＋素材）
```

実績一覧カードはトップ内に置き、カードを押すと詳細ページへ遷移。

---

## 2. ファイル構成（案）

```
portfolio/
├── index.html          トップLP
├── works/
│   └── detail.html     実績詳細の共通テンプレ（?id=xxx で内容切替）

├── data/
│   └── works.json          実績データ（triage から変換）
```

## 3. ワイヤーフレーム（デスクトップ / 上から下）

### ① ヒーロー
```
┌────────────────────────────────────────────┐
│  [氏名/ロゴ]                    works  about  contact │ ← 固定ヘッダ
├────────────────────────────────────────────┤
│                                            │
│   会計・バックオフィスの"面倒"を、         │ ← H1キャッチ
│   AIと自動化で"実装まで"片付けるエンジニア │
│                                            │
│   業務を理解 → 自動化・AIで実装 → 成果まで。│ ← サブコピー
│                                            │
│   [ 実績を見る ]   [ 相談する(note/SNS) ]  │ ← CTA×2
└────────────────────────────────────────────┘
```

### ② こんな"面倒"を解決します（課題ベース・ペルソナA）
```
┌────────────────────────────────────────────┐
│  こんな作業、まだ手でやっていませんか？     │
│                                            │
│  ┌───────┐ ┌───────┐ ┌───────┐            │
│  │手作業の│ │定型レポ│ │会計・経 │           │
│  │集計・転記│ │ート作成 │ │理の繰返し│          │
│  └───────┘ └───────┘ └───────┘            │
│      → AIと自動化で「実装まで」やります     │
└────────────────────────────────────────────┘
```
※カードの「面倒」3〜4個は後で実案件に合わせて確定

### ③ 3つの強み（重なりが特徴）
```
┌────────────────────────────────────────────┐
│   ①業務ドメイン理解   ②自動化・RPA   ③AI・LLM │
│   (会計/財務/BO)      (GAS/VBA)     (実務組込) │
│                                            │
│        ベン図 or 3カラムで「重なり」を表現   │
│        ＝ 現場を理解して実装まで持っていける │
└────────────────────────────────────────────┘
```

### ④ 制作実績（カード一覧・JSON駆動）
```
┌────────────────────────────────────────────┐
│  制作実績                                   │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐     │
│  │[サムネ]   │ │[サムネ]   │ │[サムネ]   │     │
│  │ツール名   │ │ツール名   │ │ツール名   │     │
│  │課題→成果1行│ │課題→成果1行│ │課題→成果1行│   │
│  │[公開コード]│ │[成果のみ] │ │[公開コード]│  ← バッジ │
│  │  詳細→    │ │  詳細→    │ │  詳細→    │     │
│  └──────────┘ └──────────┘ └──────────┘     │
└────────────────────────────────────────────┘
```

### ⑤ 技術スタック（ペルソナB）
```
┌────────────────────────────────────────────┐
│  技術スタック                               │
│  言語:    Python / JS / GAS / VBA ...       │
│  AI/LLM:  （ローカルLLM・エージェント等）   │
│  自動化:  RPA / スクリプト / API連携        │
│  その他:  Git / ...                         │
└────────────────────────────────────────────┘
```

### ⑥ 依頼の流れ（任意・安心材料）
```
┌────────────────────────────────────────────┐
│  相談 → 課題ヒアリング → 試作 → 実装 → 納品  │
└────────────────────────────────────────────┘
```

### ⑦ note記事
```
┌────────────────────────────────────────────┐
│  考え方を発信しています → [noteへ]          │
└────────────────────────────────────────────┘
```

### ⑧ プロフィール
```

│  [簡潔な自己紹介。守秘・個人情報は出さない]  │
└────────────────────────────────────────────┘
```

### ⑨ 連絡導線（フッタ）

## 4. モバイル時の挙動

- ヘッダ: ハンバーガーメニュー化、CTAは縦並び
- ②③④のカード: 3カラム → 1カラム縦積み
- 画像: 遅延読み込み（loading="lazy"）で軽量化

---

## 5. 実績データのスキーマ（works.json）

> works-triage.md の各行を、この形に変換すればカードが自動生成される。

```json
[
  {
    "id": "asset-management-ledger",
    "title": "資産管理台帳ツール",
    "summary": "手作業の固定資産集計を自動化し、月◯時間を削減",
    "tags": ["会計", "自動化", "Python"],
    "badge": "公開コード",          // "公開コード" | "成果のみ"
    "thumb": "assets/img/xxx.png",
    "problem": "（課題）",
    "action": "（対応・実装）",
    "result": "（成果・できれば数字）",
    "links": { "repo": "", "demo": "", "note": "" }
  }
]
```
