## 1. これは何の作業か

業務効率化・AI・財務系の自作ツールが散在して実力が伝わらない問題を解決するため、
実績を1か所に集約する**公開ポートフォリオ**を作る。上流工程（要件定義）から進行中。

- 最優先ゴール: **フリーランス案件の獲得**（次いで転職、名刺代わり）
- 設計の軸: **クライアント目線**（この人に頼むと何が解決するか）
- 見せる相手は2層: ペルソナA=非技術の発注者 / ペルソナB=技術者・エージェント

## 4. ★最重要の発見（次の人が必ず踏まえる点）

公開GitHub（ChRo05）には現在、**学習用リポと fork しか無い**。
強みの本体（asset-management-ledger / roi-simulator / report-hub / ai-strategy-agent /
ローカル完結LLMツール等）は**1件も第三者から見えていない**。
→ ポートフォリオの成否は「この非公開の本体をどう見せるか」に懸かっている。
これが次の作業（実績の仕分け）の動機。

## 5. 残タスク（あなた待ち＝オーナーの判断が必要）

1. ⭐ **実績の仕分け**（最優先・基本設計の前提）
   - 公開コードとして見せる 1〜2件を選ぶ（守秘に触れない範囲）
   - 成果のみでぼかす機密案件を選ぶ
2. **KPI** の件数を決める（例: 公開後3か月で問い合わせ◯件 / 面談◯件）
3. **連絡URL** を埋める（X等のSNS、note）

## 6. 次にやること（着手候補）

- (a) このフォルダを `git init` してバージョン管理開始（v0.1→v0.2…を履歴化）※未実施
- (b) ②基本設計へ: サイト構成図＋ワイヤーフレームのたたき台作成

推奨順: **5-1（実績仕分け）→ (b)基本設計**。仕分けが構成の前提になるため。
